# Project Instructions

## Development Journal

After completing every task or subtask, update `DEVELOPMENT_JOURNAL.md` in the project root.

Each entry must include:
- `## YYYY-MM-DD` date heading (reuse existing date heading if already present)
- `### Task: <short title>`
- `**Timestamp:**` — current date/time
- `**Prompt:**` — the exact user prompt that triggered the work (quoted block)
- `**Implementation:**` — bullet list of what was done, tools used, decisions made
- `**Files changed:**` — list of files created, modified, or deleted

Update the journal before the end of the turn in which the subtask is completed, not as a separate follow-up turn.
